# 結論：Kiro × Claude Codeで開発体験が劇的に向上した

AWS発のAI開発アシスタント「Kiro」と「Claude Code」を組み合わせることで、開発における「質と速度の両取り」を実現できます。私は実際に太陽系シミュレーターの開発で、この2つのツールを組み合わせた結果、**想像以上の開発体験**を得ることができました。

正直なところ、最初は「AIツール同士を組み合わせるなんて面倒では？」と思っていました。しかし実際に試してみると、それぞれの弱みを補完し合って、一つのツールでは実現できない理想的な開発体験が生まれました。

具体的には：
- **Kiro**：対話形式で詳細な仕様書を作成（質の担保）
- **Claude Code**：仕様書を基に爆速で実装（速度の実現）

この組み合わせにより、従来の「質か速度か」という二択から解放され、両方を手に入れることができます。

本記事では、実際の太陽系シミュレーター開発体験を基に、効果的な組み合わせ方法を詳しく解説します。

# Kiroとは？対話形式で詳細仕様書を作成するAI

Kiroは、AWS発のAI開発アシスタントです。最大の特徴は、**対話形式で詳細な仕様書を作成できる**ことです。

## Kiroの強み

1. **対話形式での要件定義**
   - プロジェクトの要件を対話しながら整理
   - 曖昧な部分を質問で明確化
   - 段階的に詳細度を上げていける

2. **高品質な設計書生成**
   - 技術的に正確な仕様書
   - 実装に必要な詳細まで記載
   - 構造化された見やすい形式

## Kiroの弱み（現時点）

率直に言うと、**現時点では実装速度が遅い**です。設計書作成は得意ですが、実際のコーディングになると時間がかかります。これは今後改善される可能性が高いですが、現在の制約として認識しておく必要があります。

# Claude Codeとは？爆速開発を実現するAI

Claude Codeは、Anthropic社のAIアシスタントClaude 3.5 Sonnetをベースとした開発支援ツールです。私が今まで使った開発ツールの中でも、特に**使い続けたい**と感じるツールの一つです。

## Claude Codeの強み

1. **爆速開発**
   - 明確な指示があれば非常に高速
   - 複数ファイルを同時に編集可能
   - 実装から動作確認まで一気通貫

2. **優れた開発体験**
   - レスポンシブで気持ちよく使える
   - エラー対応も迅速
   - 継続して使いたくなる体験設計

## Claude Codeの弱み

1. **指示出しの難しさ**
   - 曖昧な指示だと手戻りが発生
   - 詳細な仕様がないと期待と異なる実装になりがち
   - 要件定義段階での使用は困難

2. **コンテキストの制約**
   - 長い会話では初期の内容を忘れることがある

# 実践例：太陽系シミュレーター開発での体験

実際に太陽系シミュレーターを開発した際の体験を詳しく紹介します。

## プロジェクト概要

- **目的**：インタラクティブな太陽系シミュレーターの作成
- **技術**：HTML5 Canvas + JavaScript
- **機能**：惑星の軌道運動、ズーム機能、惑星情報表示

## Kiroでの仕様書作成プロセス

Kiroとの対話は非常にスムーズでした。以下のような流れで進みました：

1. **初期要件の共有**
   ```
   私：「太陽系シミュレーターを作りたいです」
   Kiro：「素晴らしいプロジェクトですね。どのような機能を想定していますか？」
   ```

2. **詳細の掘り下げ**
   Kiroは適切な質問を投げかけて、曖昧だった要件を明確化してくれました：
   - 「どの惑星まで表示しますか？」
   - 「軌道の正確性はどの程度必要ですか？」
   - 「ユーザー操作はどのようなものを想定していますか？」

3. **技術選択の相談**
   実装技術についても適切なアドバイスをもらえました：
   - Canvas vs SVG の選択理由
   - パフォーマンス最適化のポイント
   - 必要なライブラリの提案

## Claude Codeでの実装プロセス

Kiroで作成した詳細な設計書をClaude Codeに渡すと、実装が劇的にスムーズになりました。

### 仕様書の受け渡し方法

1. Kiroで作成した設計書をMarkdown形式でコピー
2. Claude Codeに「この設計書に従って実装してください」と依頼
3. 必要に応じて追加の要望を伝達

### 実装速度と品質

結果は想像以上でした：

- **実装時間**：設計書なしだと2-3時間かかりそうな内容が、30分程度で完成
- **コード品質**：構造化されたきれいなコード
- **機能完成度**：設計書の仕様を忠実に実装
- **感動ポイント**：「こんなにスムーズに開発できるなんて！」と率直に驚きました

実際のコードスニペット例：

```javascript
class Planet {
  constructor(name, distance, radius, speed, color, info) {
    this.name = name;
    this.distance = distance;
    this.radius = radius;
    this.speed = speed;
    this.color = color;
    this.info = info;
    this.angle = Math.random() * Math.PI * 2; // ランダムな初期位置
  }

  update() {
    this.angle += this.speed;
  }

  draw(ctx, centerX, centerY, scale) {
    const x = centerX + Math.cos(this.angle) * this.distance * scale;
    const y = centerY + Math.sin(this.angle) * this.distance * scale;
    
    // 軌道を描画
    ctx.beginPath();
    ctx.strokeStyle = 'rgba(255, 255, 255, 0.1)';
    ctx.arc(centerX, centerY, this.distance * scale, 0, Math.PI * 2);
    ctx.stroke();
    
    // 惑星を描画
    ctx.beginPath();
    ctx.fillStyle = this.color;
    ctx.arc(x, y, this.radius * scale, 0, Math.PI * 2);
    ctx.fill();
  }
}
```

# 効果的な組み合わせ方法

## ワークフロー

実際に効果的だった手順は以下の通りです：

### ステップ1: Kiroで仕様書作成（30-60分）
1. プロジェクトの概要を伝える
2. Kiroの質問に答えながら要件を明確化
3. 技術選択についてアドバイスをもらう
4. 詳細な設計書を作成してもらう

### ステップ2: 仕様書の整理と準備（5-10分）
1. 設計書をMarkdown形式で整理
2. 実装時に必要な追加情報があれば補足
3. プライオリティの明確化

### ステップ3: Claude Codeでの実装（30-90分）
1. 設計書をClaude Codeに共有
2. 「この設計書に従って実装してください」と指示
3. 必要に応じて調整や追加要望を伝達

### ステップ4: 品質確認と調整（15-30分）
1. 動作確認とテスト
2. 細かい調整や改善
3. ドキュメント更新

## 役割分担のポイント

| フェーズ | Kiro | Claude Code |
|---------|------|-------------|
| 要件定義 | ◎ 対話形式で詳細化 | △ 曖昧な要件は苦手 |
| 設計 | ◎ 高品質な設計書作成 | ○ 設計書があれば実装方針決定可 |
| 実装 | △ 時間がかかる | ◎ 爆速で高品質 |
| テスト・調整 | ○ 問題の分析は得意 | ◎ 迅速な修正対応 |

## 注意点とコツ

### 成功のためのポイント

1. **Kiroで十分に詳細化する**
   - 急がずにしっかりと要件を詰める
   - 技術的な詳細まで含める
   - 曖昧な部分は必ず質問で明確化

2. **設計書の受け渡しを丁寧に**
   - Markdown形式で構造化
   - 実装優先度を明確に
   - 追加要件があれば明記

3. **Claude Codeには明確な指示を**
   - 「設計書に従って」を明確に伝達
   - 期待する成果物を具体的に説明
   - 疑問点があれば遠慮なく質問

### よくある失敗パターンと対処法

1. **設計書が曖昧すぎる**
   - 対処：Kiroとの対話を十分に行う
   - 技術的詳細まで含める

2. **Claude Codeで手戻りが発生**
   - 対処：設計書をより詳細に作成
   - 実装前に期待値を確認

3. **ツールの切り替えタイミングが不明確**
   - 対処：各フェーズの完了基準を明確化
   - 70-80%の完成度で次のフェーズへ

# 実践ガイド：今すぐ始められる手順

## 必要な準備

1. **Kiroへのアクセス**
   - AWS アカウントが必要
   - Amazon Q Developer の有効化
   - Kiro機能の有効化

2. **Claude Codeの準備**
   - Anthropic アカウントの作成
   - Claude Code のインストール
   - 基本的な操作方法の習得

## 最初のプロジェクトでの実践手順

### 小さなプロジェクトから始める

最初は簡単なプロジェクトで流れを掴むことをお勧めします：

1. **ToDoアプリ**
2. **計算機アプリ**
3. **ストップウォッチ**

### 具体的な手順

1. **Kiroで要件定義（30分）**
   ```
   「[選んだプロジェクト]を作りたいです。
   詳細な仕様書を作成してください。」
   ```

2. **対話で詳細化（20分）**
   - Kiroの質問に丁寧に答える
   - 不明な点は率直に「わからない」と伝える
   - 技術選択についてアドバイスをもらう

3. **設計書の確認（10分）**
   - 生成された設計書を確認
   - 不足があれば追加質問

4. **Claude Codeで実装（30-60分）**
   - 設計書をコピーして共有
   - 実装を依頼
   - 動作確認と調整

## 成功のためのベストプラクティス

### Kiro活用のコツ

1. **具体的な質問を歓迎する**
   - 曖昧な回答より「わからない」の方が良い
   - Kiroの質問は開発品質向上のチャンス

2. **技術選択はKiroに相談**
   - 最新の技術動向を踏まえたアドバイス
   - プロジェクトに適した技術選択

3. **設計書は詳細に**
   - 実装で迷わないレベルまで詳細化
   - データ構造やAPIも含める

### Claude Code活用のコツ

1. **明確な指示を心がける**
   - 「設計書通りに実装」を明確に伝達
   - 期待する成果物を具体的に説明

2. **段階的に進める**
   - 一度に全機能ではなく、段階的に実装
   - 動作確認しながら進める

3. **エラーは積極的に共有**
   - エラーメッセージをそのまま伝える
   - 問題の詳細を具体的に説明

Kiro × Claude Codeの組み合わせは、現時点でも十分に実用的です：

- **開発効率の向上**：質と速度を両立

### 統合ツールの登場

将来的には、仕様書作成から実装まで一貫して行える統合ツールが登場するかもしれません。しかし、現在の技術レベルでは、各ツールの強みを活かした使い分けが重要です。
